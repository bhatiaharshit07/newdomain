## Problem

Enterprise RAG fails when it treats retrieval as a vector database problem instead of a product, security, and evaluation problem.

## Constraints

- Documents have access rules.
- Knowledge becomes stale.
- Users ask incomplete questions.
- Results need citations.
- Teams need measurable quality.

## Tradeoffs

Hybrid retrieval is more complex than vector-only search, but it is often worth it when enterprise vocabulary, acronyms, and document structure matter.

## Security

Authorization should happen before retrieval and before generation. Filtering after generation is too late.

## Future Improvements

- Query rewriting
- Domain-specific rankers
- Feedback-driven evaluation sets
- Document freshness scoring
