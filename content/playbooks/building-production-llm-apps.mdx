## Planning

Start with the job-to-be-done, not the model. Define the workflow, the user, the failure cost, and the approval path.

## Evaluation

Create evaluation examples before the first production release. They do not need to be perfect. They need to exist early.

## Runtime Design

- Model routing
- Prompt versioning
- Tool permissions
- Trace logging
- Cost controls
- Rollback plan

## Human Approval

Human review is a feature, not a failure. Approval workflows make autonomy acceptable in enterprise environments.

## Deployment Checklist

<Callout title="Before launch">
  Define success metrics, failure modes, escalation paths, and ownership before
  expanding usage.
</Callout>
